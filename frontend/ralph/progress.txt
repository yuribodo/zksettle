- A parent `pnpm-lock.yaml` lives at `/home/mario/pnpm-lock.yaml` which Next detects as a rival workspace root. Set `outputFileTracingRoot: path.join(__dirname)` in `next.config.ts` to silence the warning.

- The `ralph/` directory inside `frontend/` is excluded from tsconfig.json `exclude`; keep it out of all tsc/next build globs so ralph scripts don't pollute type-checking. Also add `ralph/**` to the `ignores` block of `eslint.config.mjs`.

- ESLint 9 uses flat config. `eslint-config-next` is still legacy extends-style, so compose it via `FlatCompat` from `@eslint/eslintrc`. Template: `compat.extends("next/core-web-vitals", "next/typescript")`.
- `next lint` is deprecated in Next 15 — the `lint` script calls `eslint .` directly instead.
- Vitest exits 1 when no test files match; keep `--passWithNoTests` on the `test` script until at least one test exists so CI stays green between stories.
- `@vitest/ui` must match the `vitest` major version exactly (pin `@vitest/ui@^2` while on vitest 2).
- `motion` is the modern package name (was `framer-motion`). The PRD calls for "motion v11" but npm currently ships v12 as the latest line — v11 is deprecated. Use `motion@^12`; the React import surface (`motion`, `useScroll`, `useInView`, etc.) is identical.
- `three`'s `@types/three` minor version must match the installed `three` minor (e.g. both 0.184.x). Pin them in lockstep; out-of-sync majors break Three's typed scene/object APIs.
