  - Copy strings with apostrophes/quotes (e.g. `"It's an SDK."`, the `// Click "Generate proof"` terminal prompt) must use paired quote types to keep the file ESLint-clean. Use double quotes for the outer string + single/escaped quotes inside, or vice versa. eslint-config-next + typescript-eslint won't object; prettier preserves the choice.
  - The SDK code block string joins an array on `\n` rather than a template literal with real newlines — this sidesteps indentation ambiguity (template literals pick up the leading spaces of the file's indentation), keeps prettier happy, and reads identically in source and runtime.
  - Place copy constants in `src/content/`, NOT `src/lib/`. `lib/` is for code utilities (format, prng, gsap, shiki helpers); `content/` is for data/strings. This separation lets downstream consumers grep for copy changes in one folder.
