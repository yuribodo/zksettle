# E2E Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Playwright E2E tests that validate integration between the Next.js frontend and the Rust/Axum backend, covering all 6 dashboard flows (health, API keys, credentials, proofs, events, usage).

**Architecture:** Playwright opens Chromium against a built Next.js app (localhost:3000) which fetches from the api-gateway (localhost:4000). The backend services run via a dedicated `docker-compose.e2e.yml` that starts only postgres + api-gateway (the only services the frontend directly talks to). The api-gateway is configured with `GATEWAY_ALLOW_OPEN_KEYS=true` so no auth is needed. The issuer-service and indexer are not started in E2E — the api-gateway will return 502 for proxied routes (`/v1/*`), and the tests validate both success and error states.

**Tech Stack:** Playwright, Next.js 15, Docker Compose, GitHub Actions

### Task 2: Create docker-compose.e2e.yml

**Files:**
- Create: `docker-compose.e2e.yml`

- [ ] **Step 1: Create the E2E-specific compose file**

Create `docker-compose.e2e.yml` at the repo root:

```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-zksettle}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:?POSTGRES_PASSWORD must be set}
      POSTGRES_DB: zksettle_gateway
    ports:
      - "5432:5432"
    volumes:
      - ./scripts/init-test-dbs.sql:/docker-entrypoint-initdb.d/10-test-dbs.sql:ro
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-zksettle}"]
      interval: 3s
      timeout: 2s
      retries: 10

  api-gateway:
    build:
      context: ./backend
      dockerfile: Dockerfile
      target: api-gateway
    ports:
      - "4000:4000"
    environment:
      GATEWAY_PORT: "4000"
      GATEWAY_UPSTREAM_URL: "http://localhost:3333"
      GATEWAY_INDEXER_URL: "http://localhost:3334"
      GATEWAY_DATABASE_URL: "postgres://${POSTGRES_USER:-zksettle}:${POSTGRES_PASSWORD:?}@postgres:5432/zksettle_gateway"
      GATEWAY_ALLOW_OPEN_KEYS: "true"
      GATEWAY_CORS_ALLOWED_ORIGINS: "http://localhost:3000"
    depends_on:
      postgres:
        condition: service_healthy
```

The `GATEWAY_UPSTREAM_URL` and `GATEWAY_INDEXER_URL` point to non-existent services — this is intentional. The api-gateway will return 502 for proxied `/v1/*` routes. The tests for API keys, usage, and health hit the gateway directly (not proxied) and will work. The credential/proof/event tests will validate error handling for the 502 responses.

### Task 9: Create GitHub Actions E2E workflow

**Files:**
- Create: `.github/workflows/e2e.yml`

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/e2e.yml`:

```yaml
name: E2E Tests

on:
  push:
    branches: [dev]
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  e2e:
    name: Playwright E2E
    runs-on: ubuntu-latest
    timeout-minutes: 20

    steps:
      - uses: actions/checkout@v4

      - name: Start E2E backend
        run: docker compose -f docker-compose.e2e.yml up -d --build

      - name: Wait for api-gateway health
        run: |
          for i in $(seq 1 30); do
            if curl -sf http://localhost:4000/health > /dev/null 2>&1; then
              echo "api-gateway is healthy"
              exit 0
            fi
            echo "Waiting for api-gateway... ($i/30)"
            sleep 5
          done
          echo "api-gateway failed to start"
          docker compose -f docker-compose.e2e.yml logs
          exit 1

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          cache-dependency-path: frontend/pnpm-lock.yaml

      - name: Install frontend dependencies
        run: pnpm install --frozen-lockfile
        working-directory: frontend

      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium
        working-directory: frontend

      - name: Build frontend
        run: pnpm build
        working-directory: frontend
        env:
          NEXT_PUBLIC_API_BASE_URL: http://localhost:4000
          NEXT_PUBLIC_SOLANA_NETWORK: devnet

      - name: Run E2E tests
        run: pnpm test:e2e
        working-directory: frontend
        env:
          CI: true
          NEXT_PUBLIC_API_BASE_URL: http://localhost:4000

      - name: Upload Playwright report
        uses: actions/upload-artifact@v4
        if: ${{ !cancelled() }}
        with:
          name: playwright-report
          path: frontend/playwright-report/
          retention-days: 14

      - name: Stop E2E backend
        if: always()
        run: docker compose -f docker-compose.e2e.yml down -v
```
